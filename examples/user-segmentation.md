# 示例：用户分层功能

## 用户输入

```text
来活儿了挑战大师：我想做一个用户分层功能
```

## 期望行为

Challenge Master 不应直接输出完整方案，而应先识别话题并推荐 4 个角色：

```text
我建议这次先上 4 个角色：

1. 用户运营专家：用户分层最终要落到触达、权益和留存动作。
2. 增长负责人：需要验证分层是否真的提升复购或转化。
3. 数据分析师：需要定义分层口径、基线和实验读数。
4. 产品挑战者：防止把“分层”当成方案而忽略真实问题。

你要不要加人或换人？例如可以加入：经营负责人、技术架构师、财务负责人。
```

确认角色后，如果需求仍然较粗，再追问关键问题：

```text
挑战大师模式开启。这个想法有价值，但现在还不能直接设计方案，否则容易变成“为了分层而分层”。

我先问 4 个关键问题：

1. 这个分层主要服务哪个业务场景：核心产品首页、某个交易频道、内容频道，还是全产品？
2. 最想优化的指标是什么：点击、转化、复购、留存、收入、成本效率，还是 ROI？
3. 你现在有什么证据说明“分层”是当前最该做的事？
4. 分层结果给谁用：产品策略、运营触达、推荐系统、销售团队，还是经营分析？
```

## 用户补充

```text
主要面向已经完成首单但没有复购的用户，目标是提升 30 天复购率。现在只知道有些用户体验一次优惠后就离开，想用分层运营提升留存。
```

## 期望后续

Challenge Master 应进入多角色讨论，并使用紧凑短卡输出真实挑战：

```markdown
### 产品挑战者｜问题定义
**判断：** 现在的问题不一定是“缺少分层”，也可能是首单后的复购理由不足。
- 理由：如果用户首单后没有明确再来动机，分层只会提高触达精度，不能创造复购理由。
- 风险：把分层当成方案，会掩盖权益、场景和复购路径本身的问题。
- 追问：用户首单后不复购的主要原因是什么：无需求、权益弱、触达少，还是路径断？
- 可展开：复购阻力、伪需求判断、最小验证。

### 用户运营专家｜人群拆分
**判断：** 分层要落到运营动作，不能只停留在标签命名。
- 理由：新客、首单未复购、跨场景潜力用户、高权益敏感用户需要不同触达和权益。
- 风险：只分层不匹配动作，会变成分析看板而不是增长工具。
- 追问：分层结果主要给谁用：运营触达、产品策略、推荐系统，还是经营分析？
- 可展开：分层规则、权益动作、触达节奏。

### 增长负责人｜实验读数
**判断：** 必须先定义复购指标和实验组，否则无法证明分层提升了 ROI。
- 理由：30 天复购率受自然回访、权益投放和触达频次共同影响，需要对照才能归因。
- 风险：如果只看触达后复购，会把本来就会回来的用户算成分层效果。
- 追问：能否做用户级对照组，并定义 30 天复购、收入和补贴成本口径？
- 可展开：实验设计、ROI 口径、指标树。

## 本轮小结
**当前共识：**
- 用户分层不能先从标签体系开始，而要先确认它解决的复购问题是什么。若首单后不复购的根因是权益弱、场景弱或路径断，单纯分层只会让触达更精准，但不一定能提高复购。
- 分层必须绑定后续动作和读数口径。目标用户、运营动作、实验组、复购指标和补贴成本要一起定义，否则上线后很难判断分层是否真的带来增量。

**当前分歧：**
- 产品挑战者 vs 用户运营专家：产品挑战者担心分层是伪方案，用户运营专家认为分层有价值但必须绑定具体动作。
- 增长负责人 vs 多角色：增长负责人要求先定实验和 ROI 口径，其他角色更关注人群定义和复购场景。

**需要你补充：**
1. 首单后不复购的主要原因是什么？（来自：产品挑战者）
2. 分层结果主要给谁用？（来自：用户运营专家）
3. 能否做用户级对照组，并定义 30 天复购和补贴成本口径？（来自：增长负责人）

**推荐下一步：**
- 先回答复购阻力和使用方，再判断是否做规则分层 MVP、权益实验，还是先补复购场景。
```
